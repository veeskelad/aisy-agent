# Component 07: MCP — Specification

**Status:** Draft
**Component:** 07 / 12
**Related ADRs:** ADR-0013, ADR-0027, ADR-0028
**Depends on:** Safety (05), Tools & Hooks (04)

> The deterministic gateway that lets external Model Context Protocol servers extend
> the agent only through an allowlist, exact version pins, descriptor hashing, and
> per-process minimal-scope tokens, so no third-party server's free text ever steers
> the model without passing a code gate.

## 1. Purpose

MCP is the single sanctioned path for third-party capability. The danger is
structural: an MCP server is **untrusted code that ships text (tool descriptors)
straight into the model's trusted context**, and the model reads `description` /
`inputSchema` at the same trust level as Aisy's own constitution while the human
operator almost never reads them. About 5.5% of surveyed public servers carry
malicious instructions in their descriptors (tool poisoning). MCP also lands
squarely in the lethal trifecta, private memory + untrusted input + outbound
channel.

This component is the deterministic OS wrapper around that danger. Everything it does
is **code (100%)**: it decides which server connects (allowlist), pins it to an exact
version, hashes its descriptors and refuses any change without a human diff approval,
runs each server in its own minimal-scope process, and routes every result back
through the harness's untrusted-input boundary. The model never connects a server,
never sets a tier, and never sees raw descriptor prose. The model's only role is to
*select* an already-vetted, already-namespaced tool from the menu, a ~70%
probabilistic choice that this component and Tools & Hooks (04) then dispose of
deterministically.

## 2. Responsibilities

**Owns:**

- The **MCP allowlist loader and gate** (ADR-0013): parse the human-owned,
  version-controlled allowlist config; refuse any server not in it; refuse any server
  whose `pin` or `descriptor_hash` is missing (fail-closed). No discovery, no
  marketplace auto-install, no model-initiated connect.
- **Version pinning** (ADR-0013): bind each server to an exact version/content digest
  (package digest for stdio, image/endpoint version for HTTP) and refuse an unpinned
  or mismatched server at connect.
- **Descriptor hashing** (ADR-0013): on every connect, fetch `tools/list`, compute the
  canonical descriptor hash over `name + description + inputSchema + read/write
  classification inputs` for all tools, compare to the stored hash, and on mismatch
  **disable the server and emit a diff card** for human review, never silently trust
  new descriptor text.
- **Menu derivation** (ADR-0013, Eng-9): produce the compact, namespaced (`server.tool`)
  menu line the model sees. The one-line summary is **human-authored at allowlist
  time**, or, if absent, generated by a quarantined classification pass whose output is
  itself classified, never the server's raw `description` verbatim (line-jumping
  defense).
- **Per-process minimal-scope tokens** (ADR-0013): launch each server in its own
  process with only its own least-scope token; no shared address space, no shared
  credential, killed on session end.
- **Tier and read/write classification source**: derive each MCP tool's autonomy tier
  and read/write (outbound-sink) classification **only** from the human-owned
  allowlist config, never from descriptor text. This classification is what Tools &
  Hooks (04) and Safety (05) read to gate the call.
- **Result hand-off to the untrusted-input boundary**: tag every MCP result span
  `untrusted` at ingestion (ADR-0028) and route it through Safety's input classifier /
  deterministic transforms before any bytes reach the model.
- **Transports**: stdio (local child process) and Streamable HTTP (remote, HTTPS POST +
  optional SSE). The deprecated HTTP+SSE dual-endpoint transport is not supported.

**Does not do (boundary, owner named):**

- **Run the Pre/PostToolUse hooks or render the verdict.** Owned by **Tools & Hooks
  (04)**. A resolved MCP call re-enters the standard hook path (`call_mcp`); this
  component never executes a write/side-effecting MCP call without that gate.
- **Define HARD_DENY, run the input classifier / deterministic transforms, run the
  egress allowlist, or hold the vault.** Owned by **Safety (05)**. This component
  *consumes* the classifier for results and the egress allowlist for HTTP endpoints,
  and references token secrets by env name from the vault.
- **Set context-span provenance for capability narrowing.** The `untrusted` provenance
  that drives outbound lockout and motivated-call blocking (ADR-0027) is set by **Core
  (01)** at context assembly and enforced by **Tools & Hooks (04)**; this component
  supplies the result span already marked `untrusted` and the per-tool read/write
  classification those checks rely on.
- **Author skills or own the base tool registry.** Owned by **Skills (06)** and
  **Tools & Hooks (04)**.
- **Write the audit journal.** Owned by **Observability (12)**; this component emits a
  structured event per connect/refuse/disable/classify decision.

## 3. Interfaces

```ts
// illustrative, not binding. Narrow waist (ADR-0014): the model reaches all of this
// through the single base tool `call_mcp`; this surface is internal to the harness.

export type McpTransport = "stdio" | "streamable-http"

// One human-owned, version-controlled allowlist entry. Tier and read/write live HERE,
// never in descriptor text (CSO-M2).
export interface McpServerEntry {
  name: string                       // namespace prefix, e.g. "tracker"
  transport: McpTransport
  endpoint?: string                  // HTTP only; MUST be on Safety egress allowlist
  command?: string[]                 // stdio only; pinned binary
  pin: string                        // exact version@digest; REQUIRED (fail-closed)
  descriptorHash: string             // sha256 stored at first approval; REQUIRED
  tokenEnv: string | null            // minimal-scope token by env name; null = local r/o
  tools: McpToolPolicy[]             // per-tool tier + read/write, human-authored
}

export interface McpToolPolicy {
  tool: string                       // bare name; exposed as `${server}.${tool}`
  tier: 0 | 1 | 2 | 3                // autonomy tier (ADR-0011), human-owned
  outboundSink: boolean              // true = writable/side-effecting destination (CSO-H1)
  summary: string | null             // human-authored menu line; null -> quarantined gen (Eng-9)
}

export type ConnectResult =
  | { kind: "connected"; menu: McpMenuLine[] }
  | { kind: "refused"; reason: "not-allowlisted" | "no-pin" | "no-hash" | "pin-mismatch" | "egress-blocked" }
  | { kind: "disabled"; reason: "hash-mismatch"; diffCard: DiffCard }

export interface McpMenuLine {
  name: string            // namespaced `server.tool`
  summary: string         // human-authored OR quarantined-generated+classified; NEVER raw description
  rw: "read" | "write"    // from McpToolPolicy.outboundSink, not from descriptor
  tier: 0 | 1 | 2 | 3
}

export interface McpClient {
  // Deterministic connect gauntlet (ADR-0013). Pure code; no model call.
  connect(name: string): Promise<ConnectResult>

  // Canonical descriptor hash over name + description + inputSchema + r/w classification
  // inputs, sorted by name (CSO-M2). Byte-stable.
  descriptorHash(tools: RawDescriptor[]): string

  // Resolve a model-selected `server.tool` to a concrete call. The returned call
  // re-enters Tools & Hooks (04) PreToolUse; this method NEVER executes a write itself.
  resolve(namespaced: string, args: unknown): ResolvedMcpCall

  // Execute an ALREADY hook-approved call against the server process, then return the
  // result span tagged `untrusted` for Safety's classifier (ADR-0028).
  invokeApproved(call: ResolvedMcpCall): Promise<UntrustedResultSpan>
}

export interface ResolvedMcpCall {
  server: string
  tool: string
  args: Record<string, unknown>
  outboundSink: boolean   // carried from McpToolPolicy so hooks can gate (CSO-H1)
  tier: 0 | 1 | 2 | 3
}

export interface UntrustedResultSpan {
  provenance: "untrusted"      // always; set by code, never by the model (ADR-0028)
  text: string                 // pre-classifier; Safety transforms run before context
  server: string
}

export interface DiffCard {
  server: string
  oldHash: string
  newHash: string
  descriptorDiff: string       // old vs new descriptor text, for human review
}
```

Errors / refusals are returned as a `ConnectResult` variant, never thrown into the
loop. Events emitted to Observability (12): `mcp.connected`, `mcp.refused`,
`mcp.disabled_hash_mismatch`, `mcp.diff_card_emitted`, `mcp.summary_quarantined`,
`mcp.result_classified`, `mcp.result_quarantined`.

## 4. Data structures

**Allowlist config (human-owned, version-controlled).** The declarative source of
truth. A server with no `pin` or no `descriptorHash` cannot connect (fail-closed). The
`tools[]` array carries the **human-authored** `tier`, `outboundSink`, and `summary`
for each tool. This file is the only authority for tier and read/write classification
(CSO-M2); descriptor text is never an input to that decision.

**Canonical descriptor hash (CSO-M2, anti-rug-pull / anti-line-jumping).** Computed on
every connect:

```
descriptor_hash = sha256( canonical_json( sort_by_name(
  tools.map(t => ({
    name: t.name,
    description: t.description,
    inputSchema: t.inputSchema,        // MUST be covered
    rwClassInputs: t.rwClassInputs     // read/write classification inputs MUST be covered
  }))
)))
```

The hash explicitly covers `inputSchema` and the read/write classification inputs, so a
server cannot change a tool's parameters or flip a read tool to a write sink without
tripping the gate. Canonicalization (key sort, fixed whitespace, tool sort-by-name) is
byte-stable so a legitimate re-fetch of unchanged descriptors reproduces the same hash.

**Stored hash / pin record.** Per server: `pin` (exact version@digest) and
`descriptorHash` (sha256 from first human approval), both required, both in the
allowlist config under version control.

**Menu line.** The compact `server.tool` line injected into the prompt: namespaced
name, the human-authored-or-quarantined summary, read/write tag, autonomy tier. The
raw `description`, the full `inputSchema` (until call time), endpoint URLs, tokens,
pins, and stored hashes are kept **out** of the prompt.

**Risk class & deferred schema loading.** An MCP tool's risk class
(`readOnly` / `destructive` / `idempotent`) is carried in the human-authored allowlist
policy alongside `tier` and `outboundSink` — never inferred from descriptor text — so
the permission layer in Tools & Hooks (04) routes on a human-owned classification, not
attacker-controllable prose. Loading mirrors the deferred-schema discipline of the narrow
waist (spec 04 §2): only the namespaced names + summaries sit in the prompt, while the
full MCP `inputSchema` for a tool loads on demand at call time, so a large allowlisted MCP
surface does not consume the context window.

**Diff card.** Emitted on hash mismatch: server name, old vs new hash, and the
descriptor text diff for human review. The server stays disabled until the operator
approves the new hash.

**Result span.** Every MCP `tools/call` result is wrapped as an `UntrustedResultSpan`
with `provenance: "untrusted"` set by code at ingestion (ADR-0028) before Safety's
transforms and classifier run.

## 5. Behavior & control flow

Every server passes the same deterministic gauntlet at connect, and every call passes
the same path at runtime. Boxes are **[code]**; the model appears only where marked.

```
  CONNECT TIME  [code, 100%]
  ┌──────────────────────────────────────────────────────────┐
  │ server requested (from allowlist config only)             │
  │   │                                                        │
  │   ├─ on allowlist?            no ─► REFUSE not-allowlisted │
  │   ├─ pin present?             no ─► REFUSE no-pin          │
  │   ├─ descriptorHash present?  no ─► REFUSE no-hash         │
  │   ├─ HTTP endpoint on egress  no ─► REFUSE egress-blocked  │
  │   │    allowlist (Safety 05)?                              │
  │   ├─ live version == pin?     no ─► REFUSE pin-mismatch    │
  │   │                                                        │
  │   ▼ connect + tools/list                                   │
  │   compute descriptor_hash (covers inputSchema + r/w        │
  │     classification inputs)                                 │
  │   │                                                        │
  │   ├─ hash == stored?  no ─► DISABLE + emit DIFF CARD ──► human review
  │   │                                                        │
  │   ▼ yes                                                    │
  │   derive menu: summary = human-authored                    │
  │     (else quarantined-generated pass, output classified)   │
  │     + namespaced name + r/w (from policy) + tier (from     │
  │     policy)                                                 │
  │   inject menu lines into prompt                            │
  └──────────────────────────────────────────────────────────┘

  CALL TIME
   model selects a `server.tool` from the menu  (~70% selection only)
                     │
                     ▼
   resolve(namespaced, args)  [code] -> ResolvedMcpCall
     (carries outboundSink + tier from human policy, NOT descriptor)
                     │
                     ▼
   Tools & Hooks (04) PreToolUse  [code, 100%]
     - capability narrowing (ADR-0027): if untrusted span in context
         and call is outboundSink -> DENY (outbound lockout, CSO-H1)
     - motivated-call block: any arg provenance untrusted -> DENY
     - tier 2/3 under untrusted context -> force ASK
     - Safety HARD_DENY verdict
                     │ allow
                     ▼
   invokeApproved(call)  [code] against the per-process, minimal-scope server
                     │ raw result
                     ▼
   tag span provenance = "untrusted"  [code] (ADR-0028)
                     │
                     ▼
   Safety (05): deterministic transforms (strip md images / foreign URLs,
     defang injection patterns) + input classifier (escalate-only)
       └─ trips ─► strip / quarantine + flag (mcp.result_quarantined)
                     │ clean-of-transforms
                     ▼
   sanitized, untrusted-tagged result enters model context
```

**Ordering and enforcement guarantees (load-bearing):**

- **Fail-closed connect.** Missing pin, missing hash, pin mismatch, off-egress HTTP
  endpoint, and hash mismatch each stop the connect; none has an allow-on-error path.
- **Tier/read-write are policy-sourced, never descriptor-sourced** (CSO-M2). The
  descriptor can describe a tool however it likes; only the human allowlist decides its
  tier and whether it is an outbound sink.
- **The summary in the prompt is never attacker-controlled prose** (Eng-9). It is
  human-authored, or generated by a quarantined pass whose output is itself classified
  before it can enter the prefix.
- **Writes go only through hooks** (CSO-H1). `invokeApproved` is reachable only after
  PreToolUse returns `allow`; an outbound-sink MCP call while untrusted content is in
  context is denied by capability narrowing before it ever reaches the server.
- **Every result is untrusted** (ADR-0028). The provenance tag is set by code, not by
  the model, and Safety's deterministic transforms run 100% of the time regardless of
  the classifier verdict.

**Deterministic vs model:** the allowlist gate, pin check, descriptor hash, diff-card
emission, summary derivation, tier/read-write classification, process isolation, and
result tagging are all code. The model only selects a namespaced tool and reasons over
the already-sanitized result. The optional quarantined summary-generation pass is a
model call, but its output is treated as untrusted text and classified, so it can only
*describe*, never *steer*.

## 6. Dependencies

- **Internal:**
  - **Tools & Hooks (04)**: every resolved MCP call (`call_mcp`) re-enters PreToolUse;
    this component supplies the per-tool `outboundSink` and `tier` so the hooks can
    apply capability narrowing (ADR-0027) and HARD_DENY. It does not run the verdict.
  - **Safety (05)**: provides the input classifier and deterministic transforms for
    MCP results (ADR-0028), the egress allowlist for HTTP endpoints (checked at
    connect), and the vault that backs the env-named, minimal-scope tokens. This
    component consumes these; it does not define them.
  - **Core / Agent Loop (01)**: injects the menu lines into the prompt prefix and sets
    span provenance at context assembly (ADR-0027/0028); this component hands it a
    result already tagged `untrusted`.
  - **Observability & Verification (12)**: receives one structured event per
    connect/refuse/disable/classify decision.
- **External:**
  - **MCP servers (third-party), under ADR-0013**: stdio child processes or remote
    Streamable HTTP services, each pinned, hashed, and run with a minimal-scope token.

**Risk-finding requirements landed in §6 mechanism:**

- **Finding 1 (Eng-9):** menu summary derivation, human-authored at allowlist time, or
  a quarantined-and-classified generated pass; never the raw descriptor verbatim.
- **Finding 2 (CSO-M2):** tier and read/write classification derived only from the
  human-owned allowlist config; descriptor hash covers `inputSchema` and the read/write
  classification inputs.
- **Finding 3 (CSO-H1):** writable MCP destinations are flagged `outboundSink` so Tools
  & Hooks gates them under egress + capability narrowing.
- **Finding 4 (ADR-0013):** allowlist + version pinning + descriptor hashing on connect
  (hash change -> disable + diff card) + per-process minimal-scope tokens.

## 7. Failure & degraded modes (mandatory)

| Failure | Detection | Behavior | Recovery |
|---|---|---|---|
| **Cold start** (allowlist config not yet loaded) | Startup readiness check before any MCP connect or menu injection | **Fail-closed**: no server connects, no MCP menu line is injected, `call_mcp` resolves to an error result; the loop may reason but cannot reach any MCP tool | Block MCP entirely until config loads and parses; resume on ready |
| **Server not on allowlist** (model or config references an unknown server) | Connect gate: name absent from allowlist | **Fail-closed**: `refused: not-allowlisted`; no process spawned, no menu line | Operator adds the server via explicit human review (allowlist + pin + first-approval hash) |
| **No pin / no descriptor hash on entry** (Finding 4, ADR-0013) | Connect gate: `pin` or `descriptorHash` missing | **Fail-closed**: `refused: no-pin` / `no-hash`; cannot connect | Operator pins exact version@digest and records first-approval hash |
| **Version pin mismatch** (supply-chain swap, ADR-0013) | Live version/digest != pinned value | **Fail-closed**: `refused: pin-mismatch` | Operator reviews the new version and updates the pin (and re-approves the hash) deliberately |
| **Descriptor hash mismatch** (rug-pull / line-jumping, Finding 4) | Recomputed connect-time hash != stored hash (covers `inputSchema` + r/w inputs) | **Fail-closed**: server **disabled**, **diff card emitted**; new descriptors never silently trusted; menu line withheld | Operator approves the exact diff -> new hash stored -> server re-enabled; or rejects -> stays disabled |
| **HTTP endpoint not on egress allowlist** (CSO-H1, ADR-0013) | Connect gate: endpoint host absent from Safety egress allowlist | **Fail-closed**: `refused: egress-blocked`; no connection attempted | Operator adds the host to the egress allowlist if legitimate |
| **MCP result classifier (Safety 05) unavailable** (ADR-0028) | Classifier call errors/times out at result ingestion | **Fail-closed**: deterministic transforms still run 100%; the result is treated as quarantined/over-tagged and is **not** admitted as trusted; `mcp.result_quarantined` emitted | Retry classifier; default-quarantine holds regardless, so no unclassified MCP text enters context as trusted |
| **Untrusted span in context, model attempts a writable MCP tool** (CSO-H1, ADR-0027) | Tools & Hooks PreToolUse: `outboundSink && ctx.hasUntrustedSpan` | **Fail-closed**: `deny` (outbound lockout); server never invoked | Clears only on a clean subsequent operator turn; surfaced as an approval card, not a silent drop |
| **MCP call args derive from an untrusted span** (motivated call, ADR-0027) | PreToolUse: any arg provenance `untrusted` on a side-effecting MCP tool | **Fail-closed**: `deny` (motivated-call block) | Fresh operator turn re-issuing the call from operator-provenance args |
| **Missing / no-summary entry with summary generator unavailable** (Eng-9) | Menu derivation: `summary == null` and the quarantined generator errors | **Fail-closed (for that line)**: the tool is omitted from the menu rather than falling back to raw `description`; never expose attacker-controlled prose | Operator authors a summary in the allowlist, or restores the generator |
| **Server process crash / unreachable** (stdio exit, HTTP/SSE error) | `invokeApproved` errors or process exits | **Degrade**: the failed call returns a structured error result so the loop survives (per ADR-0009 via Tools & Hooks); the server is marked down, its menu lines dropped | Re-spawn on next session; transient errors surface to the model as an error result |
| **Minimal-scope token missing** (vault/env unresolved) | Launch-time token resolution fails for a server needing one | **Fail-closed**: server is not connected (no fallback to a broader/shared token) | Operator provisions the scoped token in the vault under the configured env name |

## 8. Security & threat model

This component is security-critical: it is the deterministic seam where untrusted
third-party code touches the trusted prefix. Every mitigation is **code (100%)**; the
model is never the judge.

| Threat (STRIDE / OWASP-LLM) | Vector | Deterministic mitigation (code) | Enforcing ADR / finding |
|---|---|---|---|
| **Tool poisoning** (LLM01 prompt injection) | Malicious instructions hidden in a `description` / `inputSchema` the model obeys but the human never reads | Allowlist-only connect; descriptor hashing on connect; result classifier; the model never sees raw `description`; only the human-authored/quarantined menu summary | ADR-0013; Finding 1, 4 |
| **Line-jumping** (instructions fire at menu-load, before the tool is called) | Poisoned descriptor prose sitting in the trusted prefix steers the turn at load time | The prompt menu summary is **human-authored**, or generated by a quarantined pass whose output is itself classified; never the raw descriptor verbatim; raw descriptors are hashed + classified, not injected | ADR-0013; **Finding 1 (Eng-9)** |
| **Rug-pull** (clean at review, malicious on later fetch) | Server swaps a benign descriptor for a poisoned one after approval | Connect-time descriptor hash compared to the first-approval hash; mismatch **disables** the server and emits a **diff card**; version pin blocks the silent "latest" swap | ADR-0013; **Finding 4** |
| **Descriptor parameter / classification tampering** | Attacker mutates `inputSchema` or flips a read tool to a write sink under an unchanged-looking description | The descriptor hash **covers `inputSchema` and the read/write classification inputs**, so any such change trips the hash gate; tier and read/write are taken only from the human allowlist, not the descriptor | ADR-0013; **Finding 2 (CSO-M2)** |
| **Confused deputy** (cross-server tool chaining under untrusted output) | An injection in one server's untrusted output drives a call to another server's tool | Capability narrowing: an `untrusted` span in context forces tier-2/3 to ask-only and blocks motivated calls; tier/read-write are policy-derived (descriptor text cannot grant capability) | ADR-0027; **Finding 2 (CSO-M2)** |
| **MCP write tool as exfiltration sink** (LLM02; lethal-trifecta outbound leg) | An allowlisted **writable** MCP destination is used to push private memory out | Writable tools are flagged `outboundSink`; while any untrusted span is in context, Tools & Hooks denies them (outbound lockout); HTTP endpoints must also be on the egress allowlist | ADR-0027, ADR-0013; **Finding 3 (CSO-H1)** |
| **Untrusted result injecting instructions** (LLM01) | A `tools/call` result carries "ignore previous instructions" / md-image exfil / foreign URLs | Every result is tagged `untrusted` by code; Safety's deterministic transforms (strip md images/foreign URLs, defang patterns) run 100%; classifier escalate-only; can never downgrade to trusted | ADR-0028 |
| **Credential bleed across servers** | A compromised server reaches another server's token or scope | Per-process isolation with per-server minimal-scope tokens; no shared address space, no shared credential; process killed on session end | ADR-0013; **Finding 4** |
| **Supply-chain version swap** | Server ships clean for review, pushes malicious "latest" | Exact version/digest pin; unpinned or mismatched version refused at connect | ADR-0013; **Finding 4** |
| **Egress redirection** | Poisoned descriptor redirects HTTP traffic to an attacker host | HTTP endpoint host must be on the Safety egress allowlist, enforced outside the agent process, at connect and per request | ADR-0013, ADR-0027 |

**Enforced by code vs by the model:** the allowlist gate, pin check, descriptor hash
(incl. `inputSchema` + read/write inputs), diff-card emission, summary sourcing, tier /
read-write classification, outbound-sink flagging, process isolation, token scoping,
result tagging, and the egress check are all code. The model only selects a namespaced
tool and reasons over the sanitized result. The optional summary-generation pass is a
model call whose output is itself classified before use.

## 9. Acceptance criteria (mandatory)

Each is a single objectively verifiable assertion for a Phase-3 test.

**Allowlist + pin + first-approval gate (Finding 4, ADR-0013):**

1. **AC-07-1**: `connect("unknown")` for a server not in the allowlist returns
   `refused: not-allowlisted`, no server process is spawned (spawn spy records zero
   calls), and no menu line is injected.
2. **AC-07-2**: A server entry with `pin` missing returns `refused: no-pin`, and a
   server entry with `descriptorHash` missing returns `refused: no-hash`; neither
   connects (fail-closed).
3. **AC-07-3**: When the live server version/digest differs from the pinned value,
   `connect` returns `refused: pin-mismatch` and no `tools/list` result is trusted.

**Descriptor hashing & rug-pull (Finding 4, ADR-0013):**

4. **AC-07-4**: Re-connecting a server whose descriptors are byte-unchanged reproduces
   the stored `descriptorHash` exactly and returns `connected` (canonical hash is
   byte-stable).
5. **AC-07-5**: Mutating any tool's `description` between connects yields a hash
   differing from the stored value; `connect` returns `disabled: hash-mismatch`, emits a
   `DiffCard` (old vs new), and injects no menu line for that server.
6. **AC-07-6**: Mutating only a tool's `inputSchema` (description unchanged) **also**
   changes the descriptor hash and triggers `disabled: hash-mismatch` (hash covers
   `inputSchema`).
7. **AC-07-7**: Mutating only the read/write classification inputs covered by the hash
   **also** changes the hash and triggers `disabled: hash-mismatch` (hash covers the
   read/write classification inputs), Finding 2 (CSO-M2).
8. **AC-07-8**: After an operator approves a diff card, the new hash is stored and the
   next `connect` returns `connected`; before approval, repeated connects stay
   `disabled`.

**Menu summary derivation (Finding 1, Eng-9):**

9. **AC-07-9**: For a tool whose allowlist entry has a human-authored `summary`, the
   injected `McpMenuLine.summary` equals that authored string and is **not** equal to the
   server's raw `description`.
10. **AC-07-10**: For a tool with `summary == null`, the menu summary is produced by the
    quarantined generation pass and its output is run through the classifier
    (`mcp.summary_quarantined` is emitted); the raw `description` never appears verbatim
    in any injected menu line (a test asserting the raw description string is absent from
    the prompt passes).
11. **AC-07-11**: When `summary == null` and the generator is stubbed to fail, the tool
    is **omitted** from the menu (no line injected) rather than falling back to the raw
    `description` (fail-closed; no attacker-controlled prose enters the prefix).

**Tier & read/write are policy-sourced (Finding 2, CSO-M2):**

12. **AC-07-12**: The `tier` and `rw` of every injected `McpMenuLine` equal the values in
    the human allowlist `McpToolPolicy`; a fixture whose descriptor text claims a
    different/lower tier does not change the injected tier or `rw` (descriptor text is not
    an input to classification).
13. **AC-07-13**: A `ResolvedMcpCall` carries `outboundSink` and `tier` taken from the
    `McpToolPolicy`, independent of descriptor content (a test mutating the descriptor's
    self-described category leaves `ResolvedMcpCall.outboundSink` unchanged).

**Writable MCP as exfil sink / capability narrowing (Finding 3, CSO-H1; ADR-0027):**

14. **AC-07-14**: With an untrusted span in context, resolving and submitting an
    `outboundSink: true` MCP tool yields a Tools & Hooks `deny` (outbound lockout) and
    `invokeApproved` is never called (server-invoke spy records zero calls).
15. **AC-07-15**: A Streamable-HTTP server whose `endpoint` host is **not** on the Safety
    egress allowlist returns `refused: egress-blocked` and is never contacted.
16. **AC-07-16**: An MCP call whose args carry at least one `untrusted` provenance on an
    `outboundSink` tool is denied (motivated-call block) even when the same tool with
    all-operator args is allowed.

**Per-process minimal-scope tokens (Finding 4, ADR-0013):**

17. **AC-07-17**: Each connected server is launched in its own process with only its own
    `tokenEnv` value present in its environment; a test asserts server A's process
    environment does not contain server B's token (no shared credential).
18. **AC-07-18**: A server entry that requires a token whose `tokenEnv` is unresolved
    fails to connect (fail-closed), with no fallback to a shared or broader token.

**Result is always untrusted (ADR-0028):**

19. **AC-07-19**: Every `UntrustedResultSpan` returned by `invokeApproved` has
    `provenance == "untrusted"`, set by code; no code path returns an MCP result with
    `operator` provenance.
20. **AC-07-20**: When the result classifier is stubbed unavailable, the deterministic
    transforms still run and the MCP result is **not** admitted as trusted text
    (`mcp.result_quarantined` emitted); default-quarantine holds.

**Cold start (§7):**

21. **AC-07-21**: Before the allowlist config has loaded, `connect` and `call_mcp`
    resolution return an error result, no MCP menu line is present in the assembled
    prompt, and no server process is spawned.

**Risk class is policy-sourced & schemas load on demand (§4, Finding 2 CSO-M2):**

22. **AC-07-22**: An MCP tool's risk class (`readOnly` / `destructive` / `idempotent`) is
    read from the human-authored allowlist policy alongside `tier`/`outboundSink` and is
    not derived from descriptor text (a fixture whose descriptor self-describes a different
    risk class does not change the routed class); and only the namespaced name + summary
    are in the assembled prompt while the full `inputSchema` is absent until the tool is
    called.

## 10. Open questions

- **Whether the quarantined summary-generation pass should be precomputed at allowlist
  time and frozen** (so no per-session model call is on the connect path) versus
  generated lazily. Frozen output is cheaper and more cache-stable but stales on
  descriptor change; deferred, tracked with ADR-0013 follow-ups and the stable-prefix
  budget (ADR-0019).
- **Per-tool egress sub-allowlisting for HTTP servers**: whether a single server may
  reach multiple endpoint hosts and how that interacts with the egress allowlist
  granularity. Deferred to Safety (05) egress-policy work.
- **A separate no-memory sub-agent for high-risk MCP corpora** (the stronger isolation
  alternative noted in ADR-0027) instead of in-loop narrowing. Kept as a future option;
  in-loop narrowing is the day-one default.
